Plugin monorepo for [Noctalia 5+](https://noctalia.dev). Each subdirectory is a plugin loaded directly from source — no build step.

## Plugins

| Plugin | Entry types | Description |
|--------|-------------|-------------|
| [deepseek-monitor](deepseek-monitor/) | widget + panel + service | Monitor and switch DeepSeek router modes (direct/opencode) |
| [rbw-launcher](rbw-launcher/) | launcher_provider | Search Bitwarden vault entries via rbw CLI (prefix `/rbw`) |

## Structure

```
├── catalog.toml            # Aggregated metadata (auto-generated)
├── CODE_OF_CONDUCT.md
├── CONTRIBUTING.md
├── noctalia.d.luau         # Luau types for Noctalia API (autocomplete)
├── .luaurc                 # luau-lsp config
├── .vscode/settings.json   # LSP setup
└── <plugin>/
    ├── plugin.toml         # Plugin manifest (required)
    ├── widget.luau         # Bar widget (optional)
    ├── panel.luau          # Floating panel (optional)
    ├── service.luau        # Background service (optional)
    ├── launcher.luau       # Launcher provider (optional)
    ├── assets/             # SVG/PNG images
    ├── translations/en.json
    └── spec/*.allium       # Allium behavioral specs
```

## Adding a New Plugin

1. Create `new-plugin/plugin.toml` with metadata + entry declarations
2. Create entry files (`widget.luau`, `panel.luau`, `service.luau`, `launcher.luau` as needed)
3. Optionally add `translations/en.json`, `spec/*.allium`, `assets/`
4. Add `.allium-loop/` to root `.gitignore` (auto-generated by Allium)
5. Regenerate catalog: follow instructions in `catalog.toml` header

## Development

```bash
# Validate plugin manifest
noctalia validate-plugin deepseek-monitor
noctalia validate-plugin rbw-launcher

# Allium spec checks
allium check spec/monitoring.allium

# Luau static analysis (via CI)
luau-analyze --defs noctalia.d.luau rbw-launcher/launcher.luau
```

Plugins are loaded from the repo path configured in Noctalia settings. `.luau` files hot-reload on save (service may need full restart).

## Editor Setup

`.vscode/settings.json` and `noctalia.d.luau` provide autocomplete for the Noctalia API via [luau-lsp](https://github.com/JohnnyMorganz/luau-lsp). Types are loaded from `noctalia.d.luau` automatically.

## Dependencies

- **Noctalia 5.0.0+** — runtime
- **Luau** — language (subset of Lua 5.1 with type annotations)
- **Allium** (optional) — behavioral spec language
- **curl** — used by services for HTTP polling (`noctalia.http` unavailable in service context)
